# Контракт D-Bus — `org.klippa.Daemon1`

Канонический источник истины для обеих сторон (расширение ↔ демон). Машиночитаемая
версия — [`dbus-interface.xml`](dbus-interface.xml). При изменении правятся **оба** файла.

| Параметр | Значение |
|---|---|
| Шина | session bus |
| Well-known имя | `org.klippa.Daemon` |
| Путь объекта | `/org/klippa/Daemon` |
| Интерфейс | `org.klippa.Daemon1` |

Доступ ограничен сессией пользователя (session bus не выходит за её пределы).

## Методы

### `Capture(s kind, ay data, s source_app, x ts, b sensitive) → ()`
Расширение присылает новое содержимое буфера.
- `kind` — `"text"` или `"image"`.
- `data` — для `text` UTF-8 байты; для `image` PNG-байты.
- `source_app` — `wm_class` активного окна на момент копирования (для возможного исключения по приложению; может быть пустым).
- `ts` — Unix-время (секунды), int64.
- `sensitive` — `true`, если буфер помечен менеджером паролей (concealed/secret MIME). При `skip_secrets=true` демон такую запись **не сохраняет**.

Демон: декодирует/валидирует, отбрасывает пустое и превышающее лимиты, выполняет dedup/MRU, вытесняет старейшее сверх `max_entries`, эмитит `HistoryChanged`.

### `GetHistory(u limit) → (s json)`
Метаданные до `limit` последних записей, новейшая первой. `limit=0` — все доступные.
`json` — массив объектов:
```json
[{ "id": 42, "kind": "text", "preview": "первая строка…", "has_image": false, "ts": 1769300000 }]
```
Тяжёлые данные (полный текст, картинки) сюда не входят — берутся отдельными вызовами.

### `GetThumbnail(t id) → (ay png)`
PNG-миниатюра для `image`-записи (генерируется в памяти, на диск не пишется). Для `text`-записи — пустой массив.

### `GetContent(t id) → (s kind, ay data)`
Полное содержимое для вставки. Семантика `kind`/`data` — как в `Capture`.

### `Promote(t id) → ()`
Поднять запись в начало истории (MRU). Вызывается при выборе перед вставкой.

### `Delete(t id) → ()`
Удалить одну запись (из памяти и БД).

### `Clear() → ()`
Очистить всю историю — память и зашифрованную БД (паника).

### `GetConfig() → (s json)`
Текущий конфиг демона как JSON-объект со всеми ключами `config.toml`:
```json
{ "hotkey": "<Super>v", "max_entries": 50, "capture_images": true,
  "max_image_mb": 10, "auto_paste": true, "skip_secrets": true, "expire_days": 0 }
```
Используется страницей настроек (`prefs.js`) для заполнения виджетов.

### `SetConfig(s json) → ()`
Записать конфиг (вызывается `prefs.js`). `json` — объект с подмножеством ключей (неизвестные игнорируются). Демон валидирует типы, сохраняет в `config.toml` (источник истины) и применяет на лету: пересчитывает лимит истории, зеркалит хоткей/лимит/авто-вставку в GSettings расширения. Неверный тип → ошибка `org.klippa.Daemon1.Error.BadConfig`.

## Сигналы

### `HistoryChanged()`
Эмитится при любой мутации истории (`Capture`/`Promote`/`Delete`/`Clear`). Открытый popup перечитывает `GetHistory`.

## Замечания по реализации
- Идентификаторы записей — монотонный `uint64`, не переиспользуются после удаления.
- Картинки передаются как байтовые массивы; размер ограничен `max_image_mb` (см. конфиг). При проблемах с большими сообщениями — фолбэк на передачу через файл в `$XDG_RUNTIME_DIR` (риск R-4).
- Ошибки возвращаются как стандартные D-Bus errors; расширение обязано мягко деградировать, если демон недоступен.
